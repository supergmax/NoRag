1. What is NoRag?

NoRag is a document question-answering system with no embedding layer. Where classical RAG transforms your documents into opaque mathematical vectors, NoRag keeps everything in plain, readable Markdown and delegates routing to the contextual intelligence of modern LLMs.

The architecture rests on two Markdown index files and a folder of source documents. No vector database. No embedding API. No chunking strategy to tune.

Index layout

data/
├── index.md                ← document catalog (doc_id, sections, keywords)
├── index_system_prompt.md  ← agent catalog (agent_id, capabilities, system prompt)
└── documents/
    ├── contract_saas.md    ← full text, split by ## section_id headers
    ├── cloud_architecture.md
    └── ...

index.md is the routing brain — it lists every document with its sections and keywords, compact enough to fit in a single LLM context window. The Router model reads it, picks the relevant documents and the appropriate agent, then the Answer model reads the full sections and responds with mandatory citations.

2. The L1 Pipeline — 2 calls, deterministic

L1 is the core NoRag primitive. Every request is resolved in exactly two LLM calls, no matter how complex the question.

Call 1 — Router (SLM) Reads index.md + index_system_prompt.md + the user question. Returns a structured JSON: { agent_id, documents: [{ doc_id, sections }], reasoning }. A small, fast model (e.g. Gemini 2.5 Flash Lite) handles this cheaply.
Storage read The engine extracts the selected ## section_id blocks from the relevant document files. No similarity search — pure regex extraction on known section headers.
Call 2 — Answer (LLM) Receives the selected agent's system prompt + the full extracted sections + the question. Responds in natural language with mandatory citations in the format [doc_id, section_id]. A capable model (e.g. Gemini 2.5 Pro) handles this.
L1Result { answer, citations, agent_id, tokens: { router, answer } }. Every citation is traceable to an exact section in an exact document. 
POST /query
{
  "question": "What SLA is guaranteed in the SaaS contract?",
  "mode": "L1"
}

→ {
  "answer": "The SaaS contract guarantees 99.9% uptime ...",
  "citations": [{"doc_id": "contract_saas", "section": "sla_guarantees"}],
  "agent_id": "juriste_conformite",
  "tokens": {"router": 1820, "answer": 3410}
}

The Router reads both index files in a single call, selecting both the right documents and the right agent simultaneously — keeping the total at exactly 2 LLM calls.

3. The Multi_L Pipeline — parallel, then synthesized

Multi_L fans out N independent L1 layers, runs them concurrently with asyncio.gather, then synthesizes the results through an Aggregator LLM. Use it when a single L1 pass is not enough — multi-perspective analysis, question decomposition, or corpus partitioning.

Planner (SLM) Reads index.md + question + preset → emits a JSON array of layer plans: [{ agent_id, index_scope, sub_question }]. Capped at multil_max_layers (default 3).
N × L1 in parallel Each layer plan spawns a full L1 run (Router + Answer) with its own agent, scoped index, and sub-question. All layers run concurrently. Each has a configurable timeout (multil_layer_timeout_s, default 30 s).
Aggregator (LLM) Receives all layer answers. Writes a unified synthesis that explicitly names contradictions, preserves all citations, and notes which layer produced which insight.

Four presets

A

Multi-Agent

Same question, different agents. Cross-perspective analysis in one response.

B

Decomposition

Split the question into independent sub-questions, each routed separately.

C

Multi-Corpus

Same question, different agents, different document scopes.

D

Hybrid / Auto

The Planner freely combines agents, sub-questions, and scopes. Best for open-ended complex queries.

4. NoRag vs RAG — Technical Comparison

Criterion RAG (classical) NoRag
Routing method Cosine similarity on vectors LLM semantic routing on Markdown index
Infrastructure Vector DB (Pinecone, Weaviate, pgvector…) Plain Markdown files
Embeddings ✗ Required (cost per token) ✓ None
Human readability ✗ Opaque vectors ✓ 100% — any text editor
Context given to LLM Arbitrary top-K chunks Complete named sections
Citations Approximate (chunk offset) ✓ Exact [doc_id, section]
Session memory Must be implemented separately ✓ Native via index files
Auditability Debug requires ML expertise ✓ Read the index, see the routing
Scalability ✓ Billions of documents Up to ~500 documents (index size)
Setup complexity 2–6 weeks (chunker + embedder + DB + orchestrator) 2–4 hours (or 5 min with Mode A)
Vendor lock-in High (Pinecone, Weaviate migration is costly) ✓ None — plain .md files

Routing quality by scenario

ScenarioNoRagRAG
Direct, precise question ✓ Excellent ✓ Excellent
Vague or ambiguous question ✓ Good — LLM understands intent ⚠ Medium — depends on embeddings
Cross-document question ✓ Good — LLM combines ⚠ Hard — isolated chunks
Rare technical terms ✓ Good — keywords in index ⚠ Variable — OOV embeddings
10,000+ documents ⚠ Index too large for context ✓ Scalable
Frequent document updates ✓ Fast re-index (Archivist LLM) ⚠ Expensive re-embedding

6. When to Use NoRag

Choose NoRag when you have:

Internal knowledge baseFull control, auditability, no vendor lock-in

Critical technical documentationSection-level citations, guaranteed traceability
Non-technical teamMarkdown index editable in Notion or Excel

7. Quick Start

Mode A — No-code (5 minutes)

Copy the plugin prompt from norag/plugins/claude.md (or the provider of your choice) into your LLM of choice. L1 pipeline only. No code, no server.

Mode B — API (FastAPI)

git clone https://github.com/supergmax/NoRag
cd NoRag
pip install -e ".[dev]"
export GEMINI_API_KEY=your_key

# Start the API
uvicorn api.main:app --reload

# Query L1
curl -X POST http://localhost:8000/query \
  -H "Content-Type: application/json" \
  -d '{"question": "What are the SLA terms?", "mode": "L1"}'

# Query Multi_L preset A
curl -X POST http://localhost:8000/query \
  -H "Content-Type: application/json" \
  -d '{"question": "Analyze risk across all contracts", "mode": "MultiL", "preset": "A"}'

Mode C — Claude Code skill

# Install the skill (copy ~/.claude/skills/norag/SKILL.md from the repo)
# Then in any Claude Code session:

/norag What are the SLA terms in the SaaS contract?
/norag multi_l A Analyze our contracts from all angles
/norag list
/norag agents

Adding documents

POST /ingest
{
  "doc_id": "new_contract_2026",
  "raw_text": "Full document text here..."
}

# 2. Appends a card to data/index.md
